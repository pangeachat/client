# Streaming-STT golden contract fixtures (R0-1 freeze)

Byte-identical copies of this directory live in three repositories and pin the
cross-service `user_stt` contract for the streaming-STT initiative
(`pangeachat/.github#264`). The choreographer emits it, the client re-serializes
it into the Matrix `m.audio` event, and the bot reads it back. The fixtures are
the shared source of truth so the three repos cannot silently drift.

## Files (checksum-pinned in `MANIFEST.sha256`)

| File | Producer | What it freezes |
|---|---|---|
| `choreo_response_normal.json` | choreographer `SpeechToTextResponse` | Multi-token Google result: integer-millisecond word times, 0-100 word confidences, top-level `service`. |
| `choreo_response_whisper_fabricated_confidence.json` | choreographer `SpeechToTextResponse` | Whisper path: fabricated per-word confidence `1.0` (0-1 scale, matching Google/Deepgram) converts to a truthful `100` per token; transcript-level confidence stays `100`. Fixed R0-2 — before the fix the fabricated confidence was already 0-100 and double-scaled to `10000` (defect at `word_info_to_stt.py`). |
| `choreo_response_empty.json` | choreographer `SpeechToTextResponse` | Exhausted-fallback empty response: `{"results": [], "service": ...}`. |
| `matrix_event_content_current.json` | client (pre-R0-2, legacy) | The `m.audio` event content the client emitted BEFORE R0-2: `user_stt` word times **inflated x1000** and **no `service`**. Retained as the legacy-read fixture — the fixed parser must still read these persisted events without throwing. |
| `matrix_event_content_target.json` | client (`chat.dart`, current) | The event the client emits since R0-2: **true integer-millisecond** word times, 0-100 confidence passthrough, and `service` **preserved** (`speech_to_text_response_model.dart`). |

The choreo response fixtures are generated by the real serialization code (see
the choreo emission test), not hand-authored. The Matrix-event fixtures encode
the client `toJson` shape (which drops token `length`, lemma `lang`, and the
top-level `pos`; those are out of R0-1 scope and not part of the three frozen
defects).

## Golden-fixture policy

- **Do not edit a data fixture without a contract decision.** These files freeze
  a cross-repo wire shape. A change here is a change to what choreo/client/bot
  promise each other, and it must land in all three repos together.
- **The checksum test in each repo** recomputes every data file's SHA-256 and
  asserts it against `MANIFEST.sha256`. A drifted or partially-updated copy fails
  loudly. If you intentionally change a fixture, regenerate `MANIFEST.sha256` and
  copy the whole directory to all three repos in the same change.
- **Legacy vs target.** `matrix_event_content_current.json` captures the OLD
  (pre-R0-2, defective) shape the client used to emit; it is retained as a
  legacy-read guard — the fixed parser must still read those persisted events
  without throwing. `matrix_event_content_target.json` is the corrected shape
  the client emits since R0-2. The R0-2 defect fixes have LANDED: the target
  assertions are active (not skipped), and the R0-1 characterization tripwires
  that pinned the old shape have been retired.

## The three frozen defects (R0-2 fixed all three)

1. **1000x timestamp inflation — FIXED (R0-2).** The client parse/serialize
   used to multiply server milliseconds by 1000; it now passes integer
   milliseconds through unchanged (`speech_to_text_response_model.dart`).
2. **Confidence 10000 double-scale — FIXED (R0-2).** The choreographer no
   longer double-scales word confidence: Whisper's fabricated per-word
   confidence now matches the Google/Deepgram 0-1 `WordInfo.confidence`
   contract at the source (`openai_speech_to_text.py`), so the existing 0-1 ->
   0-100 conversion in `word_info_to_stt.py` is truthful for every provider.
   Token confidence stays 0-100.
3. **Dropped `service` provenance — FIXED (R0-2).** The client (and bot) models
   used to omit the top-level `service`; they now parse and re-emit it, so the
   ASR provider is preserved end to end.
